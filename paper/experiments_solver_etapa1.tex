\section{Experiments: classical sparse solvers (Step~1)}
\label{sec:exp_solver_etapa1}

This section documents a \emph{Step~1} comparison aligned with the project roadmap: the sparse recovery block is run with two classical solvers, FISTA (penalized $\ell_1$) and SPGL1 (LASSO mode via PyLops), while holding the Phase~0 data protocol fixed.
The goal is to separate formulation quality from optimizer choice before exploring trainable solvers or external baselines.

\subsection{Protocol}
\label{sec:solver_protocol}

We reuse the Phase~0 configuration in Section~\ref{sec:exp_protocol}: ten seeds
$\{7,13,23,29,31,37,41,43,47,53\}$, $\rho \in \{0.20,0.30,0.40,0.50,0.60\}$, $N=128$, $m=\lfloor \rho N\rfloor$, Gaussian $M$, identity $\Psi$, and the same noise levels.
The implementation profile is \texttt{solver\_comparison} with \texttt{dual\_cs\_solver=True}, which evaluates
\texttt{hybrid\_fista}, \texttt{hybrid\_spgl1}, \texttt{cs\_only\_fista}, \texttt{cs\_only\_spgl1}, and \texttt{ml\_only} (weighted hybrid is disabled in this profile).

\paragraph{Sparse optimization.}
FISTA selects $\lambda$ on the validation grid
$\{10^{-4},3\!\cdot\!10^{-4},10^{-3},3\!\cdot\!10^{-3},10^{-2},3\!\cdot\!10^{-2}\}$.
SPGL1 runs in LASSO mode ($\tau>0$, $\sigma=0$) with
$\tau \in \{10^{-4},3\!\cdot\!10^{-4},10^{-3},3\!\cdot\!10^{-3},10^{-2},3\!\cdot\!10^{-2},10^{-1}\}$.
Software: \texttt{pylops} matrix operator with dense $A=M\Psi$ and the \texttt{spgl1} Python package required by the PyLops wrapper.

\paragraph{Artifacts.}
The frozen run used for tables and figures in this draft is
\texttt{outputs/solver\_comparison/runs/20260417\_114526/} (CSVs, \texttt{PROTOCOL.txt}, logs).
Figures are copied under \texttt{paper/figures/solver\_comparison/runs/20260417\_114526/} with filenames \texttt{01\_\ldots 10\_} matching the Phase~0 dashboard.

\paragraph{Reading Table~\ref{tab:solver_rmse}.}
\texttt{hybrid\_fista} matches the Phase~0 \texttt{hybrid} behavior: it improves monotonically with $\rho$ and beats \texttt{ml\_only} at every ratio.
\texttt{hybrid\_spgl1} is essentially \emph{flat} near the ml-only RMSE ($\approx 0.367$) across $\rho$, so the SPGL1 LASSO instantiation tested here does not realize the hybrid gain seen under FISTA.
\texttt{cs\_only\_fista} improves with $\rho$ but remains far above hybrid methods; \texttt{cs\_only\_spgl1} stays near $0.69$ with almost no $\rho$ sensitivity, consistent with failed reconstruction (relative $\ell_2$ near one in the logs).

\subsection{Takeaway}
\label{sec:solver_takeaway}

Under this frozen synthetic protocol, \textbf{FISTA remains the preferred classical sparse solver} for the hybrid and \texttt{cs\_only} baselines.
The tested SPGL1 configuration does not improve hybrid error relative to FISTA; the hybrid SPGL1 curve is numerically close to ml-only, while \texttt{cs\_only\_spgl1} is strongly degraded.
This supports keeping FISTA as the default classical block in subsequent roadmap steps (trainable solvers, robustness sweeps) unless a revised SPGL1 parameterization or scaling study changes the conclusion.
